## 🟠 High Priority (This Week)

### Missing Rate Limiting Between API Batches
**File:** `Code.gs`
**Lines:** 451-454, 749-752
**Risk:** While not critical for current usage (2 leagues, weekly sync), rapid API calls could trigger Yahoo's undocumented rate limits during peak usage or if more leagues are added.

**Current Code:**
```javascript
for (var i = 0; i < playerKeys.length; i += chunkSize) {
  var chunk = playerKeys.slice(i, i + chunkSize);
  var path = '/league/' + leagueKey + '/players;player_keys=' + chunk.join(',') + '/stats;type=week;week=' + week;
  var data = yahooApiRequest_(path, {});
  // No delay between batches
}
```

**Recommended Fix:**
```javascript
for (var i = 0; i < playerKeys.length; i += chunkSize) {
  var chunk = playerKeys.slice(i, i + chunkSize);
  var path = '/league/' + leagueKey + '/players;player_keys=' + chunk.join(',') + '/stats;type=week;week=' + week;
  var data = yahooApiRequest_(path, {});

  // Add 200ms delay between batches to respect rate limits
  if (i + chunkSize < playerKeys.length) {
    Utilities.sleep(200);
  }
}
```

**Why:** Conservative delays prevent 429 errors as usage scales. The retry logic will handle temporary 429s, but delays prevent them from occurring.

**Impact:** Low urgency (current usage is fine), but good defensive programming for future.

### No Caching of Expensive Lookups
**File:** `Code.gs`
**Lines:** 833-902, 1095
**Risk:** `getPlayerOwnerMap_()` fetches all rosters (expensive API call) and is called once per snapshot, but the result could be cached.

**Current Code:**
```javascript
function buildLeagueSnapshot_(league) {
  // ...
  var ownerMap = getPlayerOwnerMap_(leagueKey); // API call every time

  // Used 6+ times below:
  var qbTop = getTopPlayersForWeekAndPosition_(completedWeek, 'QB', 3, ownerMap, leagueKey);
  var wrTop = getTopPlayersForWeekAndPosition_(completedWeek, 'WR', 3, ownerMap, leagueKey);
  // etc.
}
```

**Recommended Fix:**
Use CacheService to store owner map for 10 minutes:
```javascript
function getPlayerOwnerMap_(leagueKey) {
  leagueKey = leagueKey || getLeagueKey_();

  var cache = CacheService.getScriptCache();
  var cacheKey = 'ownerMap_' + leagueKey;
  var cached = cache.get(cacheKey);

  if (cached) {
    Logger.log('[getPlayerOwnerMap_] Using cached owner map for ' + leagueKey);
    return JSON.parse(cached);
  }

  var data = yahooApiRequest_('/league/' + leagueKey + '/teams;out=roster', {});
  // ... existing parsing logic ...

  cache.put(cacheKey, JSON.stringify(ownerMap), 600); // 10 min TTL
  return ownerMap;
}
```

**Why:** Caching reduces API calls and improves performance, especially for multi-league runs or debugging.

**Impact:** Saves 1 API call per league per snapshot run (minor optimization).

### No Debug Function to Test Without Email

Add a debug function:
```javascript
/**
 * Debug helper: generates snapshot and logs to console instead of emailing
 * Use this for development/testing to avoid email spam
 */
function debugSnapshotToLog() {
  var startTime = Date.now();
  API_CALL_COUNT = 0;

  try {
    var leagues = getAllLeagues_();
    if (!leagues || !leagues.length) {
      Logger.log('[debugSnapshotToLog] No leagues found for this Yahoo account.');
      return;
    }

    var sections = [];
    var completedWeek = null;

    for (var i = 0; i < leagues.length; i++) {
      var snapshot = buildLeagueSnapshot_(leagues[i]);
      sections.push(snapshot);

      if (completedWeek === null) {
        var cw = getCurrentWeek_(leagues[i].league_key);
        completedWeek = Math.max(1, cw - 1);
      }
    }

    var output = sections.join('\n\n');
    Logger.log('=== SNAPSHOT OUTPUT ===\n' + output);

    var endTime = Date.now();
    var durationSec = ((endTime - startTime) / 1000).toFixed(2);
    Logger.log('[debugSnapshotToLog] Completed in ' + durationSec + 's. Total API calls: ' + API_CALL_COUNT);

  } catch (err) {
    Logger.log('[debugSnapshotToLog] Error: ' + err.message);
    Logger.log('[debugSnapshotToLog] Stack trace: ' + err.stack);
  }
}
```

**Why:** Faster development iteration and easier debugging without email spam.
