### Duplicated parseTeam/parseTeamScore Pattern
**File:** `Code.gs`
**Lines:** 273-301 (`parseTeam` in `getWeekMatchups_`), 379-406 (`parseTeamScore` in `getWeekTeamHighlights_`)
**Risk:** Two nearly identical inline functions parse team metadata from Yahoo API responses. They differ only in which points field they prefer (`team_projected_points` vs `team_points`).

**Current Code (getWeekMatchups_):**
```javascript
function parseTeam(teamWrapper) {
  var t = teamWrapper.team;
  var metaArray = t[0];
  var meta = {};
  if (Array.isArray(metaArray)) {
    metaArray.forEach(function (entry) {
      var keys = Object.keys(entry || {});
      if (keys.length === 1) { meta[keys[0]] = entry[keys[0]]; }
    });
  }
  var teamName = meta.name;
  var managerName = '';
  if (meta.managers && meta.managers[0]) {
    var mm = meta.managers[0].manager || meta.managers[0];
    managerName = mm.nickname || mm.manager_nickname || '';
  }
  var pointsData = t[1] && (t[1].team_projected_points || t[1].team_points || ...);
  var total = pointsData && pointsData.total ? Number(pointsData.total) : 0;
  return { team_name: teamName, manager_name: managerName, points: total };
}
```

**Recommended Fix:**
Extract a shared `parseTeamMeta_` helper:
```javascript
function parseTeamMeta_(teamWrapper, pointsField) {
  var t = teamWrapper.team;
  var meta = flattenYahooMeta_(t[0]);
  var teamName = meta.name;
  var managerName = '';
  if (meta.managers && meta.managers[0]) {
    var mm = meta.managers[0].manager || meta.managers[0];
    managerName = mm.nickname || mm.manager_nickname || '';
  }
  var pointsData = t[1] && t[1][pointsField || 'team_points'];
  var total = pointsData && pointsData.total ? Number(pointsData.total) : 0;
  return { team_name: teamName, manager_name: managerName, points: total };
}
```

**Why:** Reduces duplication and ensures consistent parsing. The Yahoo metadata flattening pattern (`Array.isArray` + single-key check) is repeated ~10 times throughout the file.

### Yahoo API Response Metadata Flattening Pattern Repeated ~10 Times
**File:** `Code.gs`
**Lines:** 113-120, 176-183, 277-283, 383-389, 494-500, 548-554, 579-586, 710-716, 793-800, 885-891, 915-921, 992-998
**Risk:** The same 6-line pattern for flattening Yahoo's array-of-single-key-objects structure is duplicated throughout the file. Any bug in this pattern would need fixing in 10+ places.

**Repeated Pattern:**
```javascript
var meta = {};
if (Array.isArray(metaArray)) {
  metaArray.forEach(function (entry) {
    var keys = Object.keys(entry || {});
    if (keys.length === 1) {
      meta[keys[0]] = entry[keys[0]];
    }
  });
}
```

**Recommended Fix:**
Extract to a reusable helper:
```javascript
function flattenYahooMeta_(arr) {
  var meta = {};
  if (Array.isArray(arr)) {
    arr.forEach(function (entry) {
      var keys = Object.keys(entry || {});
      if (keys.length === 1) {
        meta[keys[0]] = entry[keys[0]];
      }
    });
  }
  return meta;
}
```

Then replace all instances: `var meta = flattenYahooMeta_(metaArray);`

**Why:** DRY principle. A single helper is easier to maintain and test than 10+ identical code blocks.

### Unicode Character in Snapshot Output
**File:** `Code.gs`
**Lines:** 1142, 1203
**Risk:** Uses `\u0013` (Device Control 3) character as a separator between team name and points. This is likely a copy/paste artifact — should be an em dash (`—`) or similar.

**Current Code:**
```javascript
var line = (idx + 1) + '. ' + t.team_name + owner + ' \u0013 ' + t.points.toFixed(1) + ' pts';
```

**Recommended Fix:**
```javascript
var line = (idx + 1) + '. ' + t.team_name + owner + ' — ' + t.points.toFixed(1) + ' pts';
```

**Why:** `\u0013` is a non-printable control character. It may render as invisible or garbled in email clients. An em dash (`—`) or hyphen is more readable.
